Servidor binario sobre HTTP y WebSocket

Diarsaba

Esta instancia expone un protocolo binario multiplexado por dominios para operaciones de base de datos, mensajeria P2P y eventos push por WebSocket.

Endpoints

  • GET / entrega esta pagina informativa.
  • POST /do procesa un frame binario completo y devuelve otro frame binario.
  • GET /ws abre una sesion WebSocket binaria con soporte de eventos push.

Dominios

  • 0x10 DB
  • 0x20 NET
  • 0x30 AUTH
  • 0x80 EVENT

Header

Cada frame usa 8 bytes de cabecera: dominio, flags, req_id en little-endian y payload_len en big-endian.

Referencia

La documentacion operativa y la especificacion del protocolo viven en el repositorio, principalmente en README.md y docs/protocolo-binario.md.

Si estas validando el proyecto desde Windows con el codigo alojado en WSL, ejecuta Cargo dentro de WSL para evitar problemas del linker GNU sobre rutas UNC.
